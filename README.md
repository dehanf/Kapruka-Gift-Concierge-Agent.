# Kapruka Gift Concierge

An AI-powered gift concierge for [Kapruka](https://www.kapruka.com), Sri Lanka's leading gifting platform. The system understands natural language requests, remembers recipient preferences and allergies, searches a vector catalog, and produces quality-reviewed recommendations through a multi-agent reflection loop.

1. **Router** ‚Äî classifies the user's message into one or more intents and extracts structured fields (recipient, allergies, location, search query).
2. **Semantic Memory** ‚Äî persists recipient profiles (allergies, preferences, location) to disk. Updated before any search runs.
3. **Catalog Agent** ‚Äî retrieves the top matching products from the vector store, filters allergens, and generates a natural-language recommendation.
4. **Critic Agent** ‚Äî reviews the recommendation for safety, relevance, accuracy, and quality. If it fails, the catalog agent revises and the loop repeats (up to `MAX_REFLECTION_ROUNDS`).

## Setup

### 1. Install dependencies

```bash
pip install -r requirements.txt
pip install anthropic sentence-transformers numpy pyyaml
```

### 2. Configure environment variables

Create a `.env` file in the project root:

```env
QDRANT_URL=https://your-cluster.qdrant.io
QDRANT_API_KEY=your_qdrant_api_key
```

The Anthropic API key is picked up automatically from the environment:

```env
ANTHROPIC_API_KEY=your_anthropic_api_key
```

### 3. Build the catalog

Scrape Kapruka and generate `data/catalog.json`:

```bash
python services/crawl.py
```

> This uses Playwright (Chromium). Install the browser with `playwright install chromium` if you haven't already.

### 4. Ingest into Qdrant

Embed the catalog and upload to your Qdrant collection:

```bash
python services/ingest_to_qdrant.py
```

## Configuration

All parameters are in `config.yaml`. Edit this file to tune behaviour ‚Äî no code changes needed.

```yaml
claude:
  model: "claude-opus-4-5"
  max_tokens: 300

catalog_agent:
  max_reflection_rounds: 2   # how many critic-revise cycles before accepting
  search_top_k: 8            # products fetched from vector store
  max_products_after_filter: 5

short_term_memory:
  max_turns: 10              # conversation turns kept in context

lt_memory:
  embedding_model: "sentence-transformers/all-MiniLM-L6-v2"
  search_top_k: 5

qdrant:
  collection_name: "kapruka_catalog"
  embedding_dim: 384
  timeout: 30

ingest:
  catalog_path: "data/catalog.json"
  embedding_model: "all-MiniLM-L6-v2"
  batch_size: 100
```

## Usage

```python
from agents.router import Router

router = Router(customer_id="customer_001")

response = router.route("I need a birthday cake for my wife. She's allergic to nuts.")
print(response)
```

The router handles everything: updating the recipient profile, searching the catalog, and returning a reviewed recommendation.

## Key Design Decisions

- **Reflection loop** ‚Äî the critic agent enforces safety (allergen checks) and accuracy (price/stock verification) before a recommendation is returned. Controlled by `max_reflection_rounds` in `config.yaml`.
- **Preference-first ordering** ‚Äî the router always runs `PREFERENCE_UPDATE` before `SEARCH`, so a message like *"My wife is allergic to nuts, find her a cake"* saves the allergy before searching.
- **Allergen filtering** ‚Äî done in code (not in the prompt) against `name + description + specs`, so it is deterministic and not LLM-dependent.
- **Centralised config** ‚Äî `config.yaml` + `utils/config.py` mean every tunable value has one home.
- **Centralised prompts** ‚Äî all system prompts live in `utils/prompts.py` for easy iteration without touching agent logic.
